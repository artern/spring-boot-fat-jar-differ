# spring-boot-fat-jar-differ

English README. For Chinese documentation, see [README.zh-CN.md](README.zh-CN.md).

`spring-boot-fat-jar-differ` is a Java 8-targeted toolchain built with Gradle 8.14.3. It compares two Spring Boot executable archives and produces an executable patcher jar that can upgrade the installed archive in place.

The repository publishes under the `io.github.artern.spring-boot-fat-jar-differ` namespace and provides three deliverables:

1. `spring-boot-fat-jar-differ.jar`: standalone CLI that compares a baseline archive and a latest archive.
2. `patcher-<target-name>.jar`: executable runtime jar generated by the CLI or Gradle plugin.
3. `io.github.artern.spring-boot-fat-jar-differ`: Gradle plugin for Spring Boot builds.

| `differ-cli` | `io.github.artern:differ-cli` | `io.github.artern.spring-boot-fat-jar-differ:differ-cli` |

The Gradle plugin ID (`io.github.artern.spring-boot-fat-jar-differ`) and its generated plugin marker artifact remain unchanged. Only the buildscript `classpath` dependency and direct Maven coordinates need updating.

## Supported archive shapes

- Spring Boot executable JAR archives produced by `bootJar`.
- Spring Boot executable WAR archives produced by `bootWar`.
- Archives with a Spring Boot `launchScript()` preamble. The preamble is captured and upgraded as a whole binary block.

## Patch model

- `META-INF/` is treated as a whole-tree replacement area.
- `BOOT-INF/classes/` is treated as a whole-tree replacement area.
- `WEB-INF/classes/` is treated as a whole-tree replacement area.
- All other entries use file-level add, replace, and delete operations.
- The generated patch bundle stores baseline and target entry metadata plus target preamble information under `BOOT-PATCH/`.

This design favors safe reconstruction and deterministic validation over the smallest possible payload.

## Project layout

- `differ-core`: archive scanning, diff planning, patch metadata, and validation.
- `differ-cli`: CLI entrypoint that assembles the executable patcher jar.
- `patcher-runtime`: runtime template embedded into the generated patcher.
- `gradle-plugin`: publishable Gradle plugin.
- `demo-spring-boot`: real Spring Boot demo project consuming the published plugin from `mavenLocal()`.

## Build and formatting

Build the Java 8-targeted binaries:

```bash
./gradlew assembleBinaries
```

Publish the local development artifacts needed by downstream Gradle and Maven consumers:

```bash
./gradlew publishToolingToMavenLocal
```

Published coordinates:

- `io.github.artern.spring-boot-fat-jar-differ:differ-core:0.1.2`
- `io.github.artern.spring-boot-fat-jar-differ:differ-cli:0.1.2`
- `io.github.artern.spring-boot-fat-jar-differ:differ-gradle-plugin:0.1.2`

Format the repository with Spotless:

```bash
./gradlew spotlessApply
```

Spotless is configured to run in the same Java 8-oriented environment as the rest of the project. The formatter is pinned to `google-java-format` 1.7 to keep the formatting toolchain compatible with Java 8.

## Standalone CLI usage

Generate an executable patcher jar:

```bash
java -jar differ-cli/build/libs/spring-boot-fat-jar-differ.jar ./baseline.jar ./latest.jar
```

When no explicit output path is supplied, the CLI writes `patcher-<latest-archive-base>.jar` in the current working directory.

Choose a custom output path:

```bash
java -jar differ-cli/build/libs/spring-boot-fat-jar-differ.jar ./baseline.jar ./latest.jar --output ./dist/custom-patcher.jar
```

Apply the generated patcher jar:

```bash
java -jar ./patcher-my-app.jar ./current.jar ./backup
```

Patch flow:

1. Create a timestamped backup in the supplied backup directory.
2. Rebuild the target archive into a temporary file.
3. Validate entry metadata, logical areas, and launch-script preamble.
4. Replace the original archive in place.

## Gradle plugin usage

Declare the plugin through `pluginManagement` so Gradle can resolve the locally published marker and implementation artifacts:

```groovy
pluginManagement {
    plugins {
        id 'io.github.artern.spring-boot-fat-jar-differ' version '0.1.0'
    }
    repositories {
        mavenLocal()
        maven { url 'https://maven.aliyun.com/repository/gradle-plugin/' }
        maven { url 'https://maven.aliyun.com/repository/public/' }
        maven { url 'https://maven.aliyun.com/repository/spring-plugin/' }
        maven { url 'https://maven.aliyun.com/repository/spring/' }
        gradlePluginPortal()
        mavenCentral()
    }
}

plugins {
    id 'io.github.artern.spring-boot-fat-jar-differ'
}
```

Optional task configuration:

```groovy
springBootFatJarDiffer {
    latestJar = layout.buildDirectory.file('libs/my-app.jar')
    baselineJar = layout.projectDirectory.file('.gradle/spring-boot-fat-jar-differ/my-app/baseline/last-release.jar')
}
```

Leave `outputPatcher` unset unless you explicitly want to override the standard `patcher-<latest-archive-base>.jar` output name.

Run the task:

```bash
./gradlew bootDiff
```

Defaults:

- Baseline archive location: `.gradle/spring-boot-fat-jar-differ/<project-name>/baseline/last-release.<archive-extension>`
- Generated patcher location: `build/fat-jar-differ/patcher-<latest-archive-base>.jar`

If the baseline archive is missing, the task fails with a message that points to the expected cache path.

For executable WAR projects that still use legacy `buildscript` style, add `classpath "io.github.artern.spring-boot-fat-jar-differ:differ-gradle-plugin:0.1.2"` and then `apply plugin: 'io.github.artern.spring-boot-fat-jar-differ'`.

## Maven integration

Maven builds cannot apply the Gradle plugin directly. Use the published CLI artifact instead.

Maven profile named `spring-boot-fat-jar-differ`:

1. Fail clearly when the baseline archive is missing.
2. Resolve `io.github.artern.spring-boot-fat-jar-differ:differ-cli:0.1.2` from `mavenLocal()`.
3. Generate the executable patcher during `verify`.
4. Refresh the cached baseline after successful generation.

Example:

```bash
mvn verify -Pspring-boot-fat-jar-differ
```
